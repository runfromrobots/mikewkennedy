How This Site Was Built by Just Talking to Claude

A plain-English walkthrough of the tools behind this site: syncing code to GitHub, what's actually being paid for, and deploying it live on Vercel. Written for anyone curious enough to try it themselves, no coding background required.

← Back to the site

The short version

Every redesign, every case study, every button color on this site was built by describing what I wanted in plain conversation to Claude Code, Anthropic's coding assistant. I didn't write any code by hand. The loop was simple: describe what I wanted, Claude wrote the actual HTML, CSS, and JavaScript, I looked at the result and asked for changes, and we repeated that until it was right.

Three tools made the finished result possible: a place to store the code (GitHub), a way to talk to an AI that can edit it directly (Claude Code), and a service that takes that code and turns it into a live website (Vercel). Here's what each of those actually means.

GitHub, in plain English

Think of GitHub as a shared, versioned folder for a project's code. Every change gets saved as a "commit," a labeled snapshot you can always go back to if something breaks. "Pushing" a commit sends it up to GitHub, where it's stored safely and can be picked up by other tools, like the hosting service that publishes the live site.

Claude Code can create that GitHub repository, save changes to it directly, and keep a running history of every edit, all without me ever typing a git command myself. The practical upside: nothing is ever really lost, and once the repository is connected to a hosting service, new commits can turn into live updates automatically.

What this actually costs

Being upfront about the money, since this is usually the first question: building this site didn't require any special enterprise tooling. Claude Code is included with a Claude Pro subscription. Since most of the work on a personal site happens in a short, concentrated burst rather than needing constant attention, the goal going in was to get the whole build done inside a single month of that subscription, then cancel it once the work was finished.

Compare that to a recurring monthly website builder subscription (in my case, Wix), which keeps charging every month whether or not any changes are actually being made. If a site doesn't need constant hands-on tinkering, one focused month of Claude Code can replace a much longer running subscription, especially once the code lives somewhere like GitHub and Vercel, both of which are free for a personal site at this scale.

Vercel, in plain English

Vercel is a hosting service built for exactly this workflow: point it at a GitHub repository, and every time new code is pushed there, Vercel automatically rebuilds and republishes the live site. No manual uploading, no FTP, no clicking "publish" anywhere.

A few things worth knowing if you're new to it:

  • It has a generous free tier that comfortably covers a personal site like this one
  • It provisions free HTTPS (the padlock icon in the address bar) automatically, with no setup required
  • A custom domain can be pointed at a Vercel project through your domain registrar's DNS settings, and Vercel handles the certificate for it
  • Every push to the main branch triggers a new deployment, usually live within a minute or two, and Vercel keeps a history of every deployment in case something ever needs to be rolled back

If you want to try this yourself

You don't need a coding background to follow this same path. The rough sequence:

  1. Ask Claude Code to set up a project and a GitHub repository for it
  2. Describe the site you want in your own words, and let Claude write and revise the actual code
  3. Ask Claude to push the code to GitHub, something as simple as saying "push this to GitHub" works
  4. Connect that GitHub repository to a new Vercel project, either through Vercel's own dashboard or by asking Claude to walk you through the steps
  5. From then on, any time you want a change, just describe it in plain language. Once Claude pushes the update to GitHub, Vercel takes care of the rest on its own

That's really the whole idea: a conversation in, a live website out.

None of this required knowing how to code going in. It required knowing what the finished site should look and feel like, and being willing to describe that clearly, over and over, until it matched.
